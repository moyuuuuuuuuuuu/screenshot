# Windows 滚动长截图设计规格

## 1. 目标与范围

首版在 Windows 10/11 上支持浏览器和常见桌面应用的滚动长截图。用户框选一个可滚动区域后，工具自动滚动、逐帧捕获并通过视觉重叠匹配生成长图；用户可随时停止，完成后进入现有编辑器继续标注、马赛克、OCR、翻译、复制或保存。

首版采用纯视觉匹配，不读取网页 DOM，也不要求目标应用提供 UI Automation 滚动信息。游戏、视频、持续动画、受保护内容和特殊 GPU 渲染界面不承诺成功。

## 2. 用户流程

1. 用户完成普通区域框选，点击统一规格的“长截图”图标。
2. 覆盖层隐藏编辑控件，锁定选区，并在选区外显示捕获状态与停止按钮。
3. 引擎捕获首帧，向目标窗口发送小步滚轮输入，等待画面稳定后捕获下一帧。
4. 相邻帧通过灰度特征寻找垂直重叠区域，只追加新出现的图像内容。
5. 连续帧无新增内容时判定到达底部；用户也可按 `Esc` 或点击停止图标提前结束。
6. 拼接成功后恢复覆盖层并把长图载入现有编辑器。
7. 若中途匹配失败，停止自动滚动并保留已成功拼接的结果，允许用户裁剪、编辑或放弃。

`Esc` 在长截图进行中表示“停止并保留结果”，而不是直接丢弃；再次取消编辑才退出截图会话。

## 3. 系统边界与模块

- `scroll-controller`：捕获状态机、滚动节奏、停止和资源上限。
- `platform/windows/scroll`：定位目标窗口并发送滚轮输入；不向上层暴露 Windows 类型。
- `frame-stability`：对连续预览帧采样，确认滚动动画或重排已经基本停止。
- `stitcher`：灰度化、候选重叠搜索、置信度评分、裁切和累计合成。
- `static-region-detector`：识别固定顶部栏、底部栏及悬浮元素，避免在每一帧重复追加。
- `long-capture-session`：管理帧、拼接结果、进度、诊断信息与取消信号。

长截图引擎位于平台捕获层与 React 编辑器之间。React 只消费类型化进度事件和最终图像，不直接控制系统滚轮或窗口。

## 4. 捕获状态机

状态依次为：`idle`、`preparing`、`capturing`、`scrolling`、`stabilizing`、`matching`、`completed | partial | failed | cancelled`。

- 只有得到首帧后才能进入 `scrolling`。
- 每次滚动后必须经过稳定检测，不能按固定延时盲目截图。
- 用户停止、达到底部、达到资源上限或匹配失败都会终止滚动。
- 已拼接至少一帧时，异常结果为 `partial`；首帧也无法取得时才为 `failed`。
- 终止路径必须恢复鼠标/键盘状态和覆盖层可见性。

## 5. 视觉匹配与拼接

每帧仅处理用户选区。先缩小并灰度化用于匹配，原始像素仅在确定接缝后参与合成。

1. 在上一帧下半部与新帧上半部搜索候选垂直重叠。
2. 使用多条水平采样带计算结构差异，排除纯色、低纹理和动画区域。
3. 对最佳候选计算置信度；低于阈值时缩小滚动步长重试一次。
4. 匹配成功后从新帧裁掉重叠部分，只追加新增像素。
5. 对连续多帧位置不变的顶部/底部区域标记为静态区域，仅保留首次出现的版本。
6. 悬浮元素仅作为局部异常区域，不得主导整帧接缝；无法可靠消除时允许保留并提示用户裁剪。

自动到底条件包括：最佳匹配显示新增高度低于阈值、连续画面哈希基本不变，且已进行至少一次滚动。至少连续两次满足条件后完成，降低动态页面误判。

## 6. 资源与安全限制

- 默认单次最多 200 帧、60,000 像素高度或 120 秒，以最先达到者为准。
- 合成使用分块画布或临时帧存储，避免每次追加都复制完整长图。
- 捕获时屏蔽工具自身覆盖层，避免状态条进入截图。
- 用户切换活动窗口、改变选区尺寸或目标窗口消失时立即停止并保留部分结果。
- 不向网络发送滚动帧；只有用户随后主动执行 OCR/翻译时，才沿用既有隐私流程上传最终选区。

## 7. 错误与反馈

界面只使用统一图标系统，捕获状态通过进度动画、帧数/高度的无障碍文本和工具提示表达。主要错误包括：目标不可滚动、画面无法稳定、重叠匹配失败、目标窗口变化、达到资源上限和内存不足。

所有可恢复错误都展示“保留部分结果”和“放弃”操作。诊断日志只记录尺寸、耗时、置信度和错误码，不保存屏幕像素或可识别文本。

## 8. 测试与验收

- 纯函数测试：重叠搜索、置信度、静态区域、到底检测、尺寸/帧数上限。
- 状态机测试：成功、用户停止、重试后成功、部分结果、目标消失和取消清理。
- 合成基准图：普通网页、固定顶栏、固定底栏、悬浮按钮、低纹理列表和不同滚动步长。
- Windows 集成测试：Edge/Chrome、资源管理器、设置及至少一个 Electron 应用。
- 人工验收：接缝无明显重复或缺行；停止后可继续使用全部编辑工具；失败不丢失已经拼接的内容。

## 9. 后续扩展

若纯视觉模式在特定应用中稳定性不足，可增加 UI Automation 辅助获取滚动范围和位置，但视觉拼接仍作为跨应用、跨平台的默认能力。macOS 和 Linux 只需分别实现滚动输入与目标窗口生命周期接口，拼接核心保持平台无关。
