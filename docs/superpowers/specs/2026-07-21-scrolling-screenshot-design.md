# Windows 手动滚动长截图设计规格

## 1. 目标与范围

首版在 Windows 10/11 上支持浏览器和常见桌面应用的手动滚动长截图。用户框选区域并点击长截图图标后，以当前画面作为首帧；用户直接操作目标窗口向下滚动，工具通过视觉变化监听自动捕获稳定画面、识别重叠内容并拼接长图。

本设计取代原有“工具模拟滚轮自动滚动”方案。长截图引擎不再发送任何滚轮输入，也不读取网页 DOM 或依赖 UI Automation 的滚动范围。游戏、视频、持续动画、受保护内容和特殊 GPU 渲染界面不承诺成功。

## 2. 用户流程

1. 用户完成普通区域框选，点击统一规格的长截图图标。
2. 工具将当前选区作为首帧，记录选区和下方目标窗口。
3. 主覆盖层变为鼠标穿透状态，仅保留透明选区边框；用户直接滚动目标窗口。
4. 选区右侧显示独立的置顶控制窗口，包含实时长图缩略预览、停止和取消图标，不显示常驻文字。
5. 工具监听选区画面变化；每次画面稳定后捕获原始分辨率帧，并通过视觉重叠匹配追加新内容。
6. 用户向上回滚时暂停追加；重新向下滚动到已捕获尾部后继续。
7. 至少成功检测一次滚动后，连续静止 2 秒自动完成；用户也可点击停止图标立即完成。
8. 完成后关闭控制窗口、恢复覆盖层输入，并把长图载入现有编辑器。
9. 点击取消才丢弃本次长图，并恢复开始长截图前的截图和编辑状态。

## 3. 窗口与输入结构

长截图进行时使用两个独立窗口：

- `overlay`：原全屏覆盖层，切换为系统级鼠标穿透。选区内部保持透明，边框绘制在捕获范围之外，不能进入结果像素。
- `long-capture-controls`：小型、无边框、置顶且可交互的控制窗口，放在选区右侧；右侧空间不足时放到左侧。它承载实时缩略图、停止和取消图标。

不能只用 CSS `pointer-events: none` 实现穿透，因为 WebView 所属原生窗口仍可能拦截目标应用的滚轮和鼠标输入。平台层必须切换原生窗口的忽略鼠标事件状态；独立控制窗口用于保留可点击操作。

控制窗口遵循现有图标规范：20×20 SVG、1.8px 描边、圆角端点、36×36 点击区域。缩略图显示当前已拼接内容；匹配异常时叠加警告图标，通过工具提示和无障碍标签提供文字说明。

## 4. 模块边界

- `manual-capture-session`：状态机、静止完成计时、停止、取消和资源限制。
- `region-observer`：定时采样选区低分辨率灰度图，识别静止、滚动和目标变化。
- `frame-stability`：确认滚动动画或页面重排已经停止，决定何时获取原始帧。
- `stitcher`：重叠搜索、方向判断、固定元素去重、裁切和累计合成。
- `capture-preview`：生成低成本缩略图，并向控制窗口发送进度事件。
- `platform/window-input`：记录目标窗口、切换覆盖层鼠标穿透、创建和定位控制窗口。

React 编辑器只发起会话并接收进度及最终图像。视觉监听和原始像素捕获在 Rust 层执行；拼接算法保持平台无关，以便后续复用于 macOS 和 Linux。

## 5. 捕获状态机

状态依次为：

```text
idle -> preparing -> observing -> scrolling -> stabilizing -> matching
     -> observing | paused_reverse | warning
     -> completed | partial | cancelled | failed
```

- `preparing`：保存原会话、记录目标、捕获首帧、创建控制窗口并启用穿透。
- `observing`：每 120ms 采样一次缩小灰度图。
- `scrolling`：画面变化超过阈值，等待内容停止移动。
- `stabilizing`：连续约 200ms 差异低于阈值后捕获原始帧。
- `matching`：计算相邻帧重叠及滚动方向。
- `paused_reverse`：识别到向上回滚，不删除已有内容；等待重新匹配已捕获尾部。
- `warning`：连续帧无法可靠匹配，保留已有结果并继续观察。

两秒静止计时只在至少成功追加过一帧后启用。滚动、页面动画或匹配恢复都会重置计时。无论以何种路径终止，会话都必须关闭控制窗口、取消鼠标穿透并恢复覆盖层状态。

## 6. 视觉监听与拼接

每次观察仅处理用户选区。低分辨率灰度图用于变化检测和候选匹配，原始像素只在画面稳定且接缝可靠时进入合成器。

1. 比较当前采样与上一稳定采样，判断画面是否变化。
2. 变化开始后持续采样，直到连续样本稳定约 200ms。
3. 捕获原始帧，并在上一有效帧的下半部与新帧上半部搜索向下滚动重叠。
4. 同时搜索反向重叠；反向结果显著更优时进入 `paused_reverse`。
5. 正向匹配成功后裁掉重叠内容，只追加新增像素。
6. 对连续多帧位置不变的顶部栏、底部栏和局部悬浮元素进行静态区域分析；置信度不足时保留像素，不误删正文。
7. 匹配失败时不结束会话；后续稳定帧继续与最后一个有效尾帧匹配。

重新向下滚动后，只要当前帧能与已捕获尾帧可靠匹配，就退出 `paused_reverse` 并恢复追加。首版不回退或删除已经拼接的内容。

## 7. 完成、错误与资源限制

- 自动完成：至少追加一帧后，连续 2 秒没有滚动或页面变化。
- 手动完成：点击停止图标，保留已成功拼接的全部内容。
- 取消：点击取消图标，丢弃长截图结果并恢复原截图会话。
- 部分结果：目标窗口关闭、选区对应屏幕发生变化、内存不足或达到资源上限时自动停止并保留有效内容。
- 失败：首帧无法捕获或基础窗口状态无法建立时，恢复原会话并显示错误。

默认单次最多 200 个有效帧、60,000 像素高度或 120 秒，以最先达到者为准。合成使用分块缓冲，避免每次追加都复制完整长图。诊断日志只记录尺寸、耗时、置信度、状态和错误码，不保存屏幕像素或可识别文本。

所有滚动帧均在本地处理。只有用户完成长截图后主动点击 OCR 或翻译，最终选区才沿用既有隐私流程上传。

## 8. 测试与验收

### 自动测试

- 状态机：开始、滚动、稳定、匹配、反向暂停、恢复、两秒完成、手动停止、取消和异常清理。
- 观察器：静态页面、滚动变化、动画噪声、稳定阈值和计时重置。
- 拼接器：正常向下、不同滚动步长、向上回滚、恢复到尾部、固定栏、悬浮按钮和低纹理页面。
- 窗口生命周期：主覆盖层穿透、控制窗口定位、终止后输入恢复以及目标窗口消失。
- 资源边界：帧数、高度、时长和部分结果。

### Windows 实机验收

- 输入方式：鼠标滚轮、触控板、Page Down/方向键和拖动滚动条。
- 内容类型：普通网页、固定顶栏、固定底栏、悬浮按钮、低纹理列表和局部动画。
- 应用：Edge/Chrome、文件资源管理器、Windows 设置及至少一个 Electron 应用。
- 结果：接缝无明显重复或缺行；侧边缩略图实时更新；向上回滚不破坏结果；完成后可继续使用标注、马赛克、OCR、翻译、复制和保存。

## 9. 后续扩展

若纯视觉观察在特定应用中资源占用过高，可增加输入事件作为采样提示，但视觉变化仍是是否捕获的最终依据。macOS 和 Linux 只需实现选区捕获、原生鼠标穿透和控制窗口接口，观察与拼接核心保持不变。
