# 长截图完成复制与遮罩生命周期设计

## 背景

长截图累计预览已经可以正确拼接，但结束阶段仍有两个问题：

1. 按 Enter 或点击完成按钮后，最终长图没有稳定写入系统剪切板。
2. 结束长截图时，四块外围原生遮罩窗口会播放 Windows 关闭动画，表现为四个黑框飞向屏幕四角。

## 目标

- Enter 与完成按钮必须走同一条完成路径。
- 最终 PNG 由原生长截图流程直接写入系统剪切板；复制成功后才退出。
- 复制失败时保留截图界面，并把具体错误返回给用户，不允许静默关闭。
- 长截图结束时不销毁四块遮罩窗口，只隐藏并在下一次会话中复用，彻底避免窗口关闭动画。
- 取消、编辑、保存、完成和错误退出都必须让遮罩立即不可见且不残留输入阻塞。

## 非目标

- 本次不改动拼接算法、OCR、翻译、普通截图输出或快捷键设置。
- 本次不实现单个异形镂空窗口；保留当前四块矩形遮罩布局。
- 本次不改变长截图预览窗口的外观和工具顺序。

## 完成与剪切板数据流

1. Enter 或完成按钮调用 `request_finish`，写入 `Finish` 动作并停止采集。
2. 拼接器生成最终 PNG。
3. 原生长截图流程在返回结果前调用统一的 PNG 剪切板输出函数。
4. 复制成功后返回“已完成且已复制”的结果；前端只关闭截图主窗口，不再重复写剪切板。
5. 复制失败时，原生命令返回包含原因的错误；主截图窗口恢复并显示错误，保留当前会话供用户重试或保存。

编辑和保存仍按现有流程处理：编辑把 PNG 返回编辑器，保存交给现有保存流程。

## 遮罩窗口生命周期

四块遮罩窗口从“每次创建并销毁”改成“按需创建、隐藏复用”：

- 首次长截图：缺少的遮罩窗口照常创建、定位、调整大小、设置透明度并显示。
- 后续长截图：读取同标签窗口，重新设置位置、尺寸、边框方向和捕获排除状态后显示。
- 任意结束路径：先取消输入阻塞所需状态，再隐藏预览窗口和四块遮罩窗口；不调用四块遮罩的 `close()`。
- 应用进程退出：由窗口系统统一释放隐藏窗口。

零尺寸边缘不显示对应窗口。下一次会话若该边缘重新出现，则重新定位并显示。

## 错误处理

- 遮罩创建、重定位、设置透明度或捕获排除失败：隐藏本次已显示的临时窗口，恢复主截图窗口并返回具体错误。
- 剪切板被占用或 PNG/DIB 输出失败：不报告完成，不关闭主截图窗口，向用户显示复制失败原因。
- 隐藏窗口失败：继续尝试处理其他窗口，并保留可诊断错误；不得用 `close()` 作为常规退路触发动画。

## 测试与验收

### 自动化测试

- Finish 动作在原生层调用剪切板写入，成功后返回完成结果。
- 原生复制失败会阻止成功完成并返回错误。
- 前端收到已原生复制的 Finish 结果时只关闭主窗口，不再次调用 `copyPng`。
- 四块遮罩在完成、取消和错误路径均执行隐藏，不执行关闭。
- 已存在的隐藏遮罩会被重新定位、调整大小并显示复用。
- 零尺寸边缘保持隐藏，不阻塞选区。

### Windows 实机验收

1. 连续滚动生成长图后按 Enter，剪切板中图片尺寸等于最终拼接尺寸。
2. 重复一次并点击完成按钮，得到相同结果。
3. 两种完成方式关闭时均看不到四个黑框或向四角飞散的动画。
4. 完成后立即再次截图，外围遮罩布局和输入阻塞仍正常。
5. Esc、编辑和保存路径关闭时同样没有黑框动画。
