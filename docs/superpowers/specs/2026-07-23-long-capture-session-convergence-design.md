# 长截图会话收敛设计

## 背景

当前长截图实现已经具备手动滚动、区域采集、重叠匹配、拼接预览、外围遮罩和侧栏控制，但 Windows 实机仍稳定出现两组故障：

1. 在浏览器、微信、资源管理器和设置等不同应用中滚动多屏后，预览仍停留在第一帧，最终图片没有增长。
2. Esc、Enter、完成按钮和关闭按钮都可能无法结束会话；侧栏按钮偶尔永久失效，退出时还可能看到黑色窗口向屏幕四角收缩。

现有实现把运动检测、候选帧匹配、会话终止和窗口清理分散在多个异步入口。采集循环会在页面仍处于滚动状态时尝试拼接，可能把不稳定画面当成已接收帧；侧栏则在原生采集循环真正结束前就永久禁用按钮。四块遮罩虽然已经改为隐藏复用，但侧栏及部分异常路径仍可能销毁原生窗口并触发 Windows 关闭动画。

本设计采用“单一原生会话协调器、稳定帧接收、窗口隐藏复用”的方案，替代继续叠加局部修补。

## 目标

- 用户手动滚动时持续检测运动，但只在画面重新稳定后执行一次重叠匹配和拼接。
- 预览、编辑、保存和剪切板始终使用同一个原生拼接结果。
- Esc、Enter、完成按钮、保存、编辑和关闭按钮统一进入原生终止协调器。
- 终止动作幂等，第一次有效动作获胜；重复按键或点击不会重复输出、重复清理或恢复旧选区。
- 任意终止动作应立即中断采集等待，并在 500 ms 内给出可见反馈。
- 长截图侧栏和四块外围遮罩只隐藏、复用，不在会话结束时销毁，避免 Windows 关闭动画和黑框。
- 清理完成前不允许启动新截图；清理完成后的下一次截图必须为空选区。
- 保留手动滚动模式、微信 4.1.11 风格、图标式界面和 `0.3` 黑色遮罩。

## 非目标

- 不恢复自动滚动或向目标应用注入滚轮事件。
- 不增加 OCR、翻译、标注或普通截图功能。
- 不改变已确认的工具栏布局、快捷键和图标方向。
- 不保证所有动态视频、游戏或持续动画内容都能自动拼接；持续变化区域需要明确警告而不是静默生成错误长图。
- 不用本次改造替换现有纵向重叠算法；先修正候选帧进入算法的时机，再依据实机证据调整阈值。

## 已批准方案

### 方案 A：会话状态机收敛

采用一个原生 `LongCaptureCoordinator` 管理完整会话。采集循环只接受稳定帧，所有终止入口只提交动作，窗口清理由唯一完成路径处理。该方案能同时消除拼接时序错误和退出竞态，改动范围可控。

### 未采用：最小修补

只把匹配移动到稳定帧并给按钮增加超时恢复，改动较小，但终止动作仍分散在全局快捷键、侧栏和主窗口中，无法保证第一次动作获胜，也无法彻底消除重复清理。

### 未采用：完整重写采集模块

将采集、运动检测、拼接和窗口管理拆成独立工作线程最彻底，但会扩大本轮回归范围。当前问题可以通过收敛现有边界解决，不需要立即重写所有算法。

## 原生会话模型

会话状态统一为：

`Idle -> Preparing -> Capturing -> MotionObserved -> Settling -> Matching -> Capturing -> Terminating -> Cleaning -> Idle`

允许从 `Preparing` 到 `Matching` 的任意活动状态直接进入 `Terminating`。错误也进入同一个终止流程，只是携带 `Failure` 结果。

协调器至少拥有：

- 单调递增的 `session_id`；
- 当前会话状态；
- 选区的物理屏幕坐标；
- 已接收的稳定尾帧；
- 拼接器及当前预览 PNG；
- 运动周期编号和当前周期的候选稳定帧；
- 首个终止动作；
- 清理是否开始、是否完成；
- 预览侧栏与四块遮罩的窗口标签；
- 不包含图像或识别文字的有界诊断事件。

所有状态变更在原生端串行化。前端不能直接修改会话状态，只能提交带 `session_id` 的动作。

## 稳定帧采集与拼接

### 首帧

准备窗口、隐藏主截图蒙层并等待合成稳定后，采集选区作为首个稳定帧。首帧直接初始化拼接器，发布 `frame_count = 1` 和初始预览。

### 运动周期

采集循环保持现有约 120 ms 的采样频率，但运动中的帧不再进入重叠匹配：

1. `MotionStarted`：进入 `MotionObserved`，为本次滚动创建新的运动周期。
2. `MotionFrame`：只更新运动状态和最新候选图，不调用拼接器。
3. 连续画面稳定达到约 200 ms：进入 `Settling`，再额外采集一帧作为稳定候选。
4. 对稳定候选与“最后一张已接收的稳定帧”执行一次纵向重叠匹配。
5. 匹配为向下滚动且新增高度大于最小阈值时，追加内容、替换稳定尾帧并发布预览。
6. 匹配为零位移时忽略该周期，不增加帧数。
7. 匹配为反向滚动时暂停追加并给侧栏显示方向提示，保留当前有效长图。
8. 匹配失败时保留当前有效长图并显示可恢复警告；下一次新的运动周期仍可继续尝试。

一个运动周期最多追加一次。新的滚动必须先产生新的 `MotionStarted`，避免同一稳定画面被重复追加。

### 恢复候选

稳定候选首次匹配失败时，可以在该运动周期内保留不超过三张低分辨率候选，仅用于重新选择更清晰的稳定帧。不得把运动中的候选直接追加到长图，也不得无限保存屏幕内容。

### 预览发布

只有拼接器成功增加高度后才发布新的预览。事件携带：

- `session_id`；
- 单调递增的 `revision`；
- `frame_count`；
- `stitched_width` 与 `stitched_height`；
- 当前预览 PNG 或现有预览资源标识；
- 可选状态提示。

侧栏丢弃旧会话或旧 `revision` 事件。编辑、保存、完成复制均从协调器取得最终拼接 PNG，不能从侧栏画面再次截图或单独重建。

## 统一终止动作

终止动作定义为：

- `Cancel`：Esc 或关闭按钮，丢弃长图并退出截图。
- `Finish`：Enter 或完成按钮，生成最终 PNG 并写入剪切板。
- `Edit`：停止采集，把最终 PNG 交给主编辑器。
- `Save`：停止采集，通过现有保存流程写入最终 PNG。
- `Failure`：采集或窗口错误，保留可诊断原因并安全退出或恢复编辑器。

全局快捷键、主窗口和侧栏都调用同一个 `request_long_capture_terminal(session_id, action)`。协调器使用一次性状态转换：

- 尚无终止动作时，记录本次动作并立即唤醒采集循环。
- 已有终止动作时，返回当前动作及进度，不覆盖第一次动作。
- 会话编号过期时返回 `stale session`，不得影响当前会话。

采集循环不能只依赖下一次定时采样发现原子标记。终止请求通过通知机制立即唤醒循环，使 Esc、Enter和按钮在目标页面不再滚动时也能快速响应。

## 侧栏交互

- 按钮提交动作后进入短暂的“处理中”状态，但不以本地 `submitted` 布尔值永久锁死。
- 侧栏根据原生返回的 `accepted / already_terminating / stale` 状态更新界面。
- 动作已接受时，侧栏立即隐藏，原生清理继续执行。
- 调用失败时恢复按钮，允许用户重试；错误原因以图标提示或现有错误区域展示。
- 侧栏窗口自身的系统关闭事件也被转换为 `Cancel`，不能绕开协调器。
- 终止完成事件携带 `session_id`，迟到事件不能关闭下一次会话。

## 窗口生命周期与清理

`scroll-capture-preview` 和四块外围遮罩全部采用“按需创建、隐藏复用”：

1. 首次使用时创建窗口。
2. 每次会话重新设置位置、尺寸、置顶、鼠标穿透和捕获排除状态。
3. 终止动作获接受后立即隐藏侧栏；进入清理时隐藏四块遮罩。
4. 常规完成、取消、保存、编辑和可恢复错误都不调用这些窗口的 `close()`。
5. 仅在应用进程真正退出时由窗口系统释放。

统一清理顺序为：

1. 阻止继续采集和预览发布。
2. 隐藏侧栏和四块遮罩。
3. 根据终止动作完成复制、保存或编辑数据交接。
4. 恢复主截图窗口的捕获排除、鼠标穿透和焦点相关状态。
5. 向主窗口发送带 `session_id` 的会话重置，清空选区、标注和长截图资源。
6. 按动作决定隐藏主窗口或恢复编辑器。
7. 注销本会话快捷键和监听器。
8. 标记清理完成并回到 `Idle`。

每一步都记录结果并继续执行后续清理，单个窗口隐藏失败不得阻塞运行时释放。清理函数必须幂等，重复调用不能产生第二次输出或窗口动画。

## 输出语义

- `Finish`：最终 PNG 成功写入系统剪切板后才报告完成；失败时返回错误并允许改用保存或编辑，不得静默丢图。
- `Save`：保存成功后退出；用户取消文件选择时返回当前会话，由界面决定继续截图或取消。
- `Edit`：恢复主编辑器并加载同一个最终 PNG，不再保留旧选区。
- `Cancel`：不输出图片，隐藏所有截图窗口并清空会话。

没有新增内容时，Finish、Save 和 Edit 仍可使用首帧；侧栏应明确显示当前为单帧结果。

## 隐私安全诊断

为定位跨应用差异，原生端保留最近若干条结构化事件：

- 时间、会话编号和状态转换；
- 运动差异比例；
- 稳定等待时长；
- 匹配方向、位移、重叠高度和置信度；
- 拼接后的宽高与帧数；
- 终止动作来源和清理步骤结果。

诊断信息不得包含截图像素、OCR 文本、窗口正文或剪切板内容。默认只在开发构建输出，并可供自动化测试断言。

## 错误处理

- 首帧捕获失败：进入 `Failure`，隐藏临时窗口并退出，不留下活动会话。
- 稳定帧匹配失败：保留已拼接结果并显示可恢复提示，不自动取消。
- 目标窗口消失或选区越界：停止采集，允许保存当前有效结果或取消。
- 剪切板写入失败：不报告 Finish 成功；保留最终 PNG 供保存或编辑。
- 窗口隐藏或状态恢复失败：继续余下清理并报告具体步骤，最终必须释放活动标记。
- 前端监听器丢失：原生协调器仍能通过 Esc/Enter和窗口关闭事件完成会话。

## 自动化测试

### Rust

- `MotionStarted` 和 `MotionFrame` 不调用重叠匹配。
- 每个运动周期只在稳定后匹配一次，成功时只追加一次。
- 零位移、反向和匹配失败不会破坏已拼接结果。
- 下一次新运动周期可从上一次警告恢复并继续追加。
- 预览 revision、帧数和高度只在成功追加时增长。
- 多个终止请求并发到达时第一次动作获胜。
- 终止通知立即唤醒采集循环。
- 清理重复执行不会重复复制、保存、发事件或销毁窗口。
- 侧栏和四块遮罩在所有终止路径均隐藏且不关闭。
- 过期会话动作和事件不能影响当前会话。

### 前端

- 侧栏提交动作后按原生状态响应，不会永久锁死。
- 请求失败后按钮恢复可用。
- 旧 session/revision 的预览与完成事件会被忽略。
- Edit、Save、Cancel 和 Finish 使用一致的终止桥接接口。
- 新截图会话重新挂载编辑器且初始选区为空。

### 回归

- 普通区域截图、标注、复制、保存、OCR、翻译和快捷键行为保持不变。
- 长截图遮罩仍覆盖完整屏幕选区外区域，黑色 alpha 为 `0.3`。
- 选区内部没有截图工具窗口阻挡滚轮。

## Windows 实机验收

在 Chrome 或 Edge、微信、资源管理器和 Windows 设置中分别执行：

1. 使用 `Alt+Shift+A` 启动截图并重新拖选可滚动区域。
2. 进入长截图，确认只出现一份预览，选区外遮罩透明度为 `0.3`。
3. 手动向下滚动至少三屏，每次停稳后预览高度和帧数均增长，最终高度大于原选区。
4. 分别使用 Enter、完成按钮、Esc 和关闭按钮结束；每种方式连续测试三次。
5. Finish 后粘贴剪切板，确认图片尺寸和预览一致。
6. Edit 和 Save 得到与预览相同的拼接内容。
7. 退出过程没有四个黑框、侧栏关闭动画或残留蒙层。
8. 退出后立即重新截图，确认必须重新拖选且所有按钮可用。

若目标页面持续播放视频或动画，应显示匹配提示，但 Esc 和关闭按钮仍必须可靠退出。

## 完成标准

- 四类应用中的三屏长截图都能得到大于一帧的有效长图。
- 所有终止入口连续三轮可靠，首次终止请求到可见窗口消失不超过 500 ms。
- 任意退出路径都不调用侧栏或遮罩的常规 `close()`，不出现黑框收缩动画。
- 完成、编辑、保存和剪切板的像素尺寸一致。
- 退出后的下一次截图为空选区。
- Rust、前端测试、类型检查、格式检查、Clippy、生产构建和 Windows debug 构建全部通过。
