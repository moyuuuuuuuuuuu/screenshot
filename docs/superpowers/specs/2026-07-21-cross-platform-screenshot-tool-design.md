# 跨平台截图工具 MVP 设计规格

## 1. 产品目标

构建一款 Windows 优先、架构可扩展至 macOS 和 Linux 的轻量截图工具。用户按下全局快捷键后，可直接在桌面覆盖层中框选区域、完成标注、复制或保存，并按需调用云端 OCR 与中英互译。

首版成功标准：

- Windows 10/11 上可稳定使用全局快捷键完成截图。
- 正确处理多显示器及 100%–200% 常见 DPI 缩放。
- 从触发截图到覆盖层可交互，在常见设备上的目标耗时不超过 300ms。
- 标注、马赛克、复制和保存无需联网。
- 只有用户主动点击 OCR 或翻译时才上传当前选区。
- 网络或云端失败不会丢失截图与编辑状态。

## 2. MVP 范围

### 包含

- Windows 系统托盘与全局快捷键。
- 全屏冻结、多显示器截图、区域框选和尺寸提示。
- 矩形、箭头、画笔、文字、马赛克。
- 撤销、重做、复制到剪贴板、保存 PNG、完成与取消。
- 在线中英文 OCR，自动判断中文或英文。
- 在线中英互译，显示原文与译文。
- Windows 浏览器及常见桌面应用的手动滚动、自动识别拼接长截图。
- 匿名设备每日额度、IP/设备限流及服务端用量统计。
- 无扣子配置时的本地开发 Mock 服务。

### 不包含

- 截图历史、固定到桌面、取色器和序号标注。
- 用户账号、同步、付费订阅和管理后台。
- 离线 OCR 或离线翻译。
- macOS/Linux 安装包；仅保留平台接口和代码边界。

## 3. 技术架构

### 桌面客户端

- Tauri 2 作为应用壳和 IPC 边界。
- React + TypeScript 负责覆盖层界面、工具栏、编辑状态与结果浮层。
- Rust 负责全局快捷键、系统托盘、屏幕捕获、显示器/DPI 信息、剪贴板与文件保存。
- HTML Canvas 负责截图预览、矢量标注交互与最终图像合成。

客户端分为以下独立模块：

1. `platform`：定义跨平台系统接口；Windows 提供首个实现。
2. `capture`：获取各显示器图像、坐标和缩放信息，生成统一虚拟桌面画布。
3. `overlay`：冻结桌面、选择区域、调整边界并处理键盘操作。
4. `editor`：维护标注对象、工具状态以及撤销/重做命令栈。
5. `renderer`：将原图、矢量标注和马赛克合成为最终位图。
6. `cloud-client`：压缩选区并调用云端 OCR/翻译 API。
7. `output`：复制 RGBA/PNG 到剪贴板或保存 PNG 文件。

React 不直接访问操作系统 API；所有系统能力都通过类型化 Tauri commands/events 调用。平台差异只存在于 Rust `platform` 实现中。

### 云端 API

首版采用轻量 TypeScript HTTP 服务，职责仅限：

- 校验匿名设备 ID、请求签名、时间戳与请求大小。
- 执行 IP 与设备维度限流、每日额度扣减。
- 调用扣子适配器或开发 Mock 适配器。
- 将供应商响应转换为稳定的客户端响应。
- 记录请求 ID、功能类型、耗时、结果状态和用量，不记录截图内容。

云端业务代码依赖 `OcrTranslationProvider` 接口，不直接依赖扣子响应结构，以便替换供应商。

## 4. 截图与编辑交互

### 主流程

1. 用户按下可配置快捷键，默认 `Alt+Shift+A`。
2. Rust 捕获所有显示器并返回虚拟桌面坐标、图像与 DPI 信息。
3. 无边框置顶覆盖层显示冻结画面，光标进入框选状态。
4. 用户拖动创建选区，之后可移动或通过八个控制点调整。
5. 工具栏紧贴选区右下方；空间不足时自动翻转到上方或左侧。
6. 用户添加标注、OCR/翻译、复制或保存。
7. Enter 等价于“复制并退出”，Esc 取消，Ctrl+S 保存。

滚动长截图采用用户手动滚动、视觉变化监听和自动重叠拼接方案，详细边界与验收标准见 `docs/superpowers/specs/2026-07-21-scrolling-screenshot-design.md`。

### 工具栏与图标规范

- 工具栏只显示图标，不显示常驻文字。
- 所有图标使用同一 SVG 线性图标系统：20×20 画布、1.8px 描边、圆角端点。
- 所有按钮点击区域固定为 36×36，工具组以细分隔线区分。
- 普通图标使用中性色；OCR/翻译为强调色；完成为绿色；取消为红色。
- 激活工具显示强调色图标与深色背景；禁用态为 40% 透明度。
- 悬停 400ms 后显示工具名称和快捷键；每个按钮提供无障碍标签。

### 编辑模型

- 矩形、箭头、画笔和文字以矢量对象保存，最终输出时再栅格化。
- 马赛克对象保存路径、笔宽和像素块尺寸；渲染时从原始截图采样，避免对已标注图层重复像素化。
- 每次创建、更新或删除对象形成可逆命令；连续画笔拖动合并为单条历史记录。
- 撤销/重做历史仅存在于当前截图会话，退出后清除。

## 5. OCR 与翻译

### 用户体验

- 点击 OCR 后上传未缩放的当前选区，结果浮层按段落显示识别文本。
- OCR 结果支持复制全部；悬停文本段时高亮对应图片区域。
- 点击翻译后执行 OCR 与语言判断，中文翻译为英文，英文翻译为简体中文。
- 翻译结果显示原文与译文，并支持复制译文和重试。
- 请求进行中允许继续查看截图，但暂时禁用重复提交同一功能。

### 扣子工作流

项目提供扣子工作流搭建文档。工作流须先在扣子平台完成试运行并发布，然后通过非流式工作流 API 调用。

工作流输入：

```json
{
  "mode": "ocr | translate",
  "image": { "file_id": "扣子文件上传接口返回的文件 ID" },
  "request_id": "uuid"
}
```

工作流输出必须是 JSON 对象：

```json
{
  "source_language": "zh | en",
  "original_text": "string",
  "translated_text": "string | null",
  "blocks": [
    {
      "text": "string",
      "confidence": 0.0,
      "box": { "x": 0.0, "y": 0.0, "width": 0.0, "height": 0.0 }
    }
  ]
}
```

`box` 坐标均为相对图片宽高的 0–1 浮点值。OCR 模式的 `translated_text` 必须为 `null`。当无法可靠判断语言时，工作流返回结构化错误，不擅自翻译。

扣子适配器先通过扣子文件上传接口上传当前选区，再把返回的 `file_id` 传给已发布工作流。文件上传和工作流执行共用同一服务端令牌，并统一受 20 秒供应商超时限制。若扣子实际工作流图片参数类型要求另一种文件引用结构，只允许在适配器内部转换，领域接口和客户端响应保持不变。

服务端配置：

- `COZE_API_BASE_URL`
- `COZE_API_TOKEN`
- `COZE_WORKFLOW_ID`
- `CLOUD_PROVIDER=coze|mock`

生产环境必须使用 `coze` 且上述三项完整，否则拒绝启动。密钥永不下发到桌面客户端。

## 6. 云端接口与数据流

### 客户端接口

- `POST /v1/ocr`
- `POST /v1/translate`
- `GET /v1/quota`

OCR 与翻译使用 `multipart/form-data`，包含压缩图片、设备 ID、请求时间戳和签名。图片最长边限制为 4096px，编码后最大 8MB；客户端优先使用 PNG，超限时改用高质量 JPEG。服务端不信任客户端声明，会重新校验格式、尺寸与实际大小。

成功响应：

```json
{
  "request_id": "uuid",
  "source_language": "zh | en",
  "original_text": "string",
  "translated_text": "string | null",
  "blocks": [],
  "quota": { "remaining": 19, "resets_at": "ISO-8601" }
}
```

设备 ID 在首次启动时随机生成，并保存在 Windows Credential Manager。签名用于降低简单脚本滥用，不作为不可伪造的设备认证。首版默认额度为每设备每天 OCR 20 次、翻译 10 次，数值由服务端配置控制。

## 7. 隐私、安全与错误处理

- 普通截图、编辑、复制和保存完全在本地完成。
- 只有点击 OCR/翻译后，客户端才上传当前选区；界面在首次使用云功能时明确提示。
- 云功能提示明确说明图片将被发送到本服务及第三方扣子平台，第三方处理遵循扣子当时生效的服务条款与隐私规则。
- 云端以内存或生命周期受控的临时文件处理图片，并在响应、失败或超时后释放。
- 日志禁止记录图片、OCR 原文、译文、访问令牌和设备签名。
- 服务端设置 IP 突发限流、设备每日额度、并发上限、8MB 请求限制及供应商超时。
- 扣子令牌仅通过服务端环境变量注入；错误响应不得包含供应商凭据或原始调试信息。

统一错误码：

- `INVALID_IMAGE`：格式、尺寸或内容不合法。
- `QUOTA_EXCEEDED`：当天额度已用完。
- `RATE_LIMITED`：请求过于频繁。
- `NETWORK_UNAVAILABLE`：客户端无法连接服务。
- `PROVIDER_TIMEOUT`：扣子工作流超时。
- `PROVIDER_INVALID_RESPONSE`：扣子输出不符合约定结构。
- `UNSUPPORTED_LANGUAGE`：未可靠识别为中文或英文。
- `INTERNAL_ERROR`：未分类的服务端错误。

所有云端错误都保留当前截图、选区和编辑历史。结果浮层显示简短状态以及重试按钮；重试复用同一截图，不重复改变编辑内容。

## 8. 状态与持久化

本地只保存必要设置：

- 全局快捷键。
- 默认保存目录。
- PNG/JPEG 输出偏好。
- 工具默认颜色、线宽和马赛克块尺寸。
- 是否已展示云功能隐私提示。

不保存截图历史、OCR 文本或译文。匿名设备 ID 存入系统凭据存储，普通设置存入 Tauri 应用配置目录。

## 9. 测试与验收

### 自动测试

- Rust 单元测试：虚拟桌面坐标、DPI 换算、选区裁剪和文件命名。
- TypeScript 单元测试：编辑命令栈、工具状态、图标可访问名称和 API 状态机。
- Canvas 快照测试：矩形、箭头、文字、画笔及马赛克合成。
- 云端测试：图片校验、额度边界、限流、扣子响应转换与敏感日志过滤。
- 契约测试：Mock 和扣子适配器必须输出相同领域模型。
- 端到端测试：快捷键后框选、标注、复制、保存，以及云端成功/超时/额度耗尽。

### Windows 手工验收矩阵

- Windows 10 与 Windows 11。
- 单显示器、横向双显示器、显示器位于主屏左侧/上方。
- 100%、125%、150%、200% 缩放以及混合 DPI。
- 快捷键冲突、剪贴板被占用、保存路径无权限。
- 断网、慢网、扣子超时、返回非法 JSON 和额度耗尽。

## 10. 交付阶段

1. 建立 monorepo、Tauri 壳、React 覆盖层和云端服务骨架。
2. 完成 Windows 截图、多显示器/DPI、选区和剪贴板输出。
3. 完成 Canvas 编辑器、统一图标工具栏和撤销/重做。
4. 完成 Mock 云端、额度/限流、OCR/翻译结果浮层。
5. 完成扣子适配器、工作流搭建文档与真实联调入口。
6. 完成安装包、自动测试和 Windows 验收。

由于当前没有扣子工作流和令牌，第 1–4 阶段可独立完成。第 5 阶段的代码与配置接口可以完成，但真实识别质量与端到端生产验证必须在用户创建并发布扣子工作流、提供服务端配置后进行。
