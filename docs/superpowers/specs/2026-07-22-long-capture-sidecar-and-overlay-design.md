# 长截图侧栏与截图蒙层修复设计

## 背景

用户实机视频暴露了三个相关问题：

1. 进入长截图后，滚轮无法传递给被截图应用。
2. 按 Esc 取消时，普通截图蒙层和长截图预览窗口分别呈现黑色并同时关闭。
3. 取消后再次使用快捷键截图，会先显示上一次的选区。

代码与视频确认了共同根因：当前长截图预览 WebView 横跨选区与侧栏，覆盖选区的区域会截获滚轮；取消操作把前端会话恢复为“已选择”而非清空；原全屏蒙层与预览窗口又按两套生命周期分别清理。

## 目标

- 进入长截图后，选区内显示被截图应用的实时内容，用户可直接手动滚动。
- 长截图预览和操作仅出现在选区外侧，不与选区发生任何像素级重叠。
- Esc 表示取消并退出：丢弃长截图结果、清空编辑会话并隐藏全部截图界面。
- 下一次快捷键截图始终从空选区开始。
- 普通截图蒙层使用单层黑色遮罩，alpha 固定为 `0.3`；选区内完全透明。
- 保留现有手动滚动识别与拼接算法，本次不调整匹配阈值。

## 非目标

- 不改回自动滚动。
- 不修改 OCR、翻译、隐私打码或普通标注工具。
- 不重写长截图拼接算法。
- 不增加可配置的蒙层透明度设置。

## 方案

采用选区外侧独立窄型侧栏（方案 A）。原全屏截图窗口在长截图期间进入鼠标穿透展示状态，只负责保留选区边界；侧栏 WebView 只承载累计预览、导航预览和四个图标操作。

### 窗口布局

- 侧栏优先放在选区右侧，空间不足时放在左侧。
- 侧栏矩形必须完全位于选区之外，二者边界可以相邻但不能相交。
- 侧栏期望宽度为 172 物理像素；可用空间不足时允许缩至 120 物理像素，并把四个图标按钮换为两行排列。高度限制在当前显示器可用范围内。
- 侧栏不再渲染覆盖选区的大型预览舞台。
- 若左右均不足 172 像素，选择空间较大的一侧并缩窄；若两侧最大可用宽度仍不足 120 像素，则明确中止长截图并恢复普通截图界面。

### 长截图状态流

1. 用户在普通截图界面提交选区并点击长截图。
2. 原截图窗口进入长截图展示状态、启用鼠标事件穿透，并从系统捕获中排除。
3. 在选区外创建侧栏窗口，并从系统捕获中排除。
4. 完成两个窗口的准备后才采集第一帧。
5. 用户在选区内滚动原应用；现有观察器识别稳定帧并交给拼接器。
6. 侧栏轮询并展示累计预览、导航预览、慢速滚动提示和操作按钮。
7. 编辑、保存、完成沿用现有输出语义；取消走统一的取消清理流程。

### 取消与清理

Esc、侧栏取消按钮和前端取消入口使用同一语义：

- 立即标记运行时取消，长截图循环停止且不产生可编辑结果。
- 先关闭侧栏，再退出原截图窗口的长截图展示状态。
- 清空前端选区、标注历史、长截图 Blob、生成的对象 URL、错误提示和服务结果。
- 将截图会话重置为 `selecting`，随后隐藏原截图窗口。
- 清理过程必须可重复调用；前端 Esc 与全局 Esc 同时到达时只执行一次有效关闭。

普通停止、编辑、保存和完成不使用上述“丢弃并重置”语义。

### 下一次截图会话

快捷键开始新截图时，在原生截图开始事件中先递增会话编号并清空前端显示状态。新桌面帧准备完成后才重新显示原截图窗口。即使窗口显示与 React 渲染发生短暂竞争，隐藏前已重置的会话也不能包含旧选区。

### 蒙层透明度

- 无选区时：整个截图区域使用 `rgba(0, 0, 0, 0.3)`。
- 有选区时：只在选区外绘制同一层 `rgba(0, 0, 0, 0.3)`，选区内部不叠加黑色遮罩。
- 移除当前全屏遮罩与选区外阴影的 alpha 叠加，避免实际视觉透明度高于 `0.3`。
- 长截图期间不显示遮罩填充，只保留必要的选区边界，从而让底层应用内容保持可读且可滚动。

## 组件边界

- `preview_windows.rs`：计算不与选区相交的侧栏矩形并创建侧栏窗口。
- `ScrollCapturePreview.tsx`：只渲染侧栏预览、提示和图标操作。
- `long_capture.rs`：编排窗口准备、捕获、终止与幂等清理顺序。
- `capture-session.ts`：区分“返回编辑”与“取消并清空会话”。
- `ScreenshotEditor.tsx`：在取消时清理编辑器本地资源，并在关闭前进入空选区状态。
- `SelectionOverlay.tsx` 与样式：实现单层 `0.3` 遮罩和长截图期间的透明展示。

## 错误处理

- 侧栏无法放在选区外、窗口穿透失败或捕获排除失败时，不开始第一帧采集。
- 准备阶段失败时关闭已创建的侧栏、恢复原窗口交互，并在普通截图界面显示具体错误。
- 取消请求不显示失败提示。
- 清理中的单个窗口调用失败不得阻止其余资源释放。

## 测试策略

### 自动化测试

- Rust 布局测试：右侧、左侧、显示器边缘和空间不足场景下，侧栏矩形均不与选区相交。
- Rust 生命周期测试：窗口准备顺序、取消清理顺序以及重复取消的幂等性。
- 前端组件测试：侧栏不再渲染覆盖选区的大型预览舞台，四个图标操作仍可用。
- 状态机测试：取消长截图后模式为 `selecting`、选区为空、长截图结果为空。
- 编辑器测试：Esc 取消只请求一次取消，关闭前清空旧选区，迟到的长截图结果被丢弃。
- 视觉指标测试：普通截图遮罩 alpha 为 `0.3`，有选区时选区内部透明。

### Windows 实机验收

1. 使用 `Alt+Shift+A` 选择可滚动页面区域并进入长截图。
2. 确认选区内可正常接收鼠标滚轮，页面内容发生滚动且长图高度增长。
3. 确认侧栏始终位于选区外，不遮挡或捕获选区内输入。
4. 按 Esc，确认侧栏与截图界面无双黑窗残影并一次退出。
5. 再次使用快捷键，确认没有旧选区，必须重新拖拽。
6. 确认普通截图选区外遮罩视觉 alpha 为 `0.3`，选区内部不变暗。

## 验收标准

- 视频中的三个问题均不可复现。
- 侧栏与选区矩形交集始终为空。
- 长截图手动滚动、编辑、保存、完成、取消行为保持可用。
- 普通截图、前端测试、Rust 测试、类型检查、构建和 Clippy 全部通过。
