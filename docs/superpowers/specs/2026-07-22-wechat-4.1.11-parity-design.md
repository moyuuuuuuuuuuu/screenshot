# Windows 微信 4.1.11 截图工具一比一重做设计规格

## 1. 目标与基准

本项目以本机安装的 Windows 微信 `4.1.11.55` 截图工具作为唯一的视觉与交互基准，重做完整截图体验，而不只调整滚动长截图。

需要复刻的范围包括全屏遮罩、窗口吸附、框选、尺寸提示、选区控制点、工具栏位置与顺序、图标、二级工具选项、标注、马赛克、隐私打码、OCR、翻译、滚动截图、撤销、保存、钉住、转发、完成、取消、快捷键和所有反馈状态。

允许存在的产品差异只有：

- 默认快捷键为 `Alt + Shift + A`，并允许用户自定义；
- OCR、翻译和隐私打码通过用户配置的扣子服务执行；
- 原生窗口与屏幕捕获层采用跨平台接口，首个完整实现仍以 Windows 10/11 为验收平台；
- 不使用微信商标、名称或品牌资源，只复刻截图工具的通用布局与交互。

设计参考来自同一台机器上实际运行的微信 4.1.11 三个状态：普通截图工具栏、滚动截图初始态、滚动并产生累计内容后的状态。网络上的旧版本资料不作为像素验收依据。

## 2. 实施策略

采用“行为优先的分层重做”。保留经过测试且可复用的底层屏幕捕获、图像编码、标注渲染、文件输出与拼接基础算法；重写截图会话、覆盖层、选区、工具栏、滚动预览和相关窗口生命周期。

React 负责微信式像素级界面与编辑交互，Rust/Tauri 负责原生窗口、鼠标穿透、屏幕捕获、目标窗口跟踪、快捷键注册和滚动帧采样。平台无关的状态机与图像算法保持纯函数或小型独立模块，以便在 macOS 和 Linux 复用。

不采用现有组件逐项换皮，因为当前滚动截图窗口结构与微信不同，换皮会保留错误的行为边界。也不采用 Windows 全原生界面重写，因为它会显著破坏跨平台目标。

## 3. 普通截图界面

### 3.1 启动与框选

- 默认通过 `Alt + Shift + A` 或托盘菜单启动截图。
- 截图开始后冻结当前桌面画面并显示全屏暗色遮罩，鼠标变为十字光标。
- 悬停窗口时识别并高亮窗口边界；拖拽时创建自由选区。
- 选区使用微信绿色细边框和八个方形控制点，支持移动和八方向缩放。
- 选区外保持半透明暗色遮罩；选区内显示冻结的原始桌面像素。
- 选区尺寸提示、光标和拖拽反馈以微信 4.1.11 为准。

### 3.2 工具栏

工具栏为白色悬浮条，使用微信式圆角、阴影、内边距和分隔线。它优先跟随选区右下方，空间不足时放到选区上方，并保持在当前显示器可见范围内。

普通截图工具顺序按实机基准固定为：

1. 矩形
2. 圆形
3. 表情
4. 箭头
5. 画笔
6. 马赛克
7. 文字
8. 扩展/隐私处理工具
9. OCR/打码入口
10. 滚动截图
11. 撤销
12. 保存
13. 钉住
14. 转发
15. 取消
16. 完成

工具栏不显示常驻文字。每个图标使用项目内维护的 SVG，不混用字符图标或不同风格的图标库。统一视觉规格为约 `18×18` 图形框、`1.6px` 描边、圆角端点和一致留白；常规图标为深灰，禁用态为浅灰，取消为红色，完成与选区为微信绿色。最终尺寸与间距以实机参考图逐像素校准。

### 3.3 标注与输出

- 矩形、圆形、箭头、画笔、文字和马赛克采用微信的默认颜色、线宽、光标及二级选项浮层。
- 切换工具不丢失已有标注；撤销只撤销最近一次编辑。
- 表情采用统一资源与微信式放置、缩放和移动方式。
- 保存写入用户选择的路径，但不立即退出截图。
- 钉住生成可移动、置顶的图片窗口。
- 转发优先调用平台分享能力；平台不支持时复制图片并给出轻提示。
- 红色取消丢弃本次会话；绿色完成复制图片并关闭截图层。

## 4. OCR、翻译与隐私打码

OCR、翻译和隐私打码沿用微信式入口、等待状态、成功反馈与失败提示，实际服务统一通过 `CozeService` 调用用户配置的扣子接口。

- 未配置扣子时，图标仍正常显示，首次点击进入配置引导。
- 普通截图、标注、保存、复制和长截图不会上传屏幕像素。
- 只有用户主动点击 OCR、翻译或自动打码后，才发送当前选区或必要的图像数据。
- OCR 结果支持复制；翻译结果以微信式浮层或结果面板展示，不破坏当前截图会话。
- 网络或服务失败时保留选区和全部标注，仅显示可重试的轻提示。

## 5. 滚动长截图

### 5.1 微信式流程

用户在普通截图工具栏点击滚动截图后：

1. 记录当前选区、首帧和原截图会话。
2. 普通工具栏消失，界面进入滚动截图专用布局。
3. 选区附近显示“滚动页面截取更多内容”提示。
4. 当前累计长图作为大预览悬浮在原选区附近，并根据累计高度自动缩放以适应可用区域。
5. 大预览右侧显示更小的整图缩略预览。
6. 控制条贴近大预览右下方，只保留编辑/返回、保存、取消、完成四个统一图标。
7. 用户直接操作目标窗口向下滚动；预览只显示已可靠拼接的内容。
8. 编辑/返回停止采集并把当前长图载入标注状态；保存保留会话；取消恢复进入长截图前的普通选区；完成输出长图并结束会话。

滚动截图界面不显示帧数、像素高度、加载文字或独立的大型侧边控制面板。

### 5.2 原生窗口结构

视觉上保持微信的单一坐标系，原生实现拆分为边界不可见的协作窗口：

- 鼠标穿透的全屏暗色遮罩与提示层；
- 大预览窗口；
- 右侧缩略图窗口；
- 可交互的四按钮控制条。

目标应用必须直接接收鼠标滚轮、触控板、滚动条拖动、方向键和 Page Down。预览及控制窗口不能覆盖用户需要滚动的选区内容，也不能进入最终截图像素。

### 5.3 连续帧运动跟踪

旧实现只在画面静止后比较两张完整截图，容易在重复内容、低纹理区域、固定栏和大步滚动中误判。新实现改为跟踪滚动期间的连续低分辨率帧轨迹：

1. 以当前首帧建立跟踪特征和已接受尾部。
2. 每个采样周期估算上一采样到当前采样的垂直位移，累计用户实际滚动轨迹。
3. 画面稳定后捕获原分辨率候选帧，用累计位移约束重叠搜索范围。
4. 在约束范围内执行精确像素/特征匹配，并分离固定顶部、固定底部与局部悬浮元素。
5. 只有置信度达标的新增区域进入拼接器；大预览和缩略图随后更新。
6. 向上滚动时暂停追加且不删除已有内容；重新回到已接受尾部后恢复。
7. 置信度不足时保留最后正确结果，提示用户缓慢滚动，不强行拼入错误帧。

采样间隔、稳定阈值和匹配阈值通过记录的合成帧序列与 Windows 实机测试校准，不能依赖单个固定魔数解决所有内容类型。

## 6. 模块边界

- `CaptureSession`：唯一截图会话状态机，管理待机、框选、标注、滚动、服务调用、完成和取消。
- `NativeOverlay`：全屏窗口、冻结桌面、鼠标穿透、窗口吸附、多显示器和 DPI 坐标。
- `WechatToolbar`：工具顺序、统一 SVG、二级选项和微信式布局。
- `AnnotationEngine`：形状、画笔、文字、马赛克、表情与撤销历史。
- `ScrollMotionTracker`：连续采样帧的垂直位移和方向估算。
- `ScrollStitcher`：受轨迹约束的重叠匹配、固定元素处理和分块合成。
- `ScrollPreview`：大预览、右侧缩略图、四按钮控制条和窗口定位。
- `CozeService`：OCR、翻译、隐私打码的配置、请求和错误映射。
- `SettingsStore`：快捷键、扣子配置和少量本机偏好。

模块通过明确的数据结构交换选区、帧、轨迹、拼接结果和进度，不允许 React 组件与平台 API 互相越层调用。

## 7. 自定义快捷键

- 默认快捷键为 `Alt + Shift + A`。
- 托盘菜单提供设置入口，允许用户录入新的组合键。
- 保存前检测格式、系统保留键和注册冲突。
- 注册失败时保留旧快捷键并显示原因。
- 注册成功后立即注销旧组合并启用新组合。
- 支持一键恢复默认值。
- 快捷键与扣子配置仅保存在本机配置目录。

## 8. 异常处理与资源清理

- 目标窗口关闭、移动到其他显示器或捕获失败时，保留当前有效长图并进入编辑状态。
- 达到帧数、高度、时长或内存限制时，保留有效结果并给出轻提示。
- OCR/翻译失败不结束截图会话。
- 快捷键冲突不覆盖仍可工作的旧配置。
- 任意成功、失败、停止、取消和异常退出路径都必须恢复鼠标输入、关闭临时窗口、注销临时监听并释放像素缓存。
- 应用崩溃后重新启动不能遗留透明遮罩或永久占用全局快捷键。

## 9. 测试与验收

### 9.1 自动测试

- 状态机：框选、标注、服务调用、滚动、编辑返回、保存、完成、取消与错误恢复。
- 选区：窗口吸附、反向拖拽、八方向缩放、跨显示器、负坐标和混合 DPI。
- 工具栏：顺序、禁用态、位置翻转、统一图标尺寸和各工具二级浮层。
- 标注：所有工具、边界裁切、撤销和最终像素输出。
- 快捷键：默认值、自定义、冲突、即时重注册和恢复默认。
- 滚动：连续位移、不同速度、正反向、重复内容、低纹理、固定栏、悬浮元素、大步滚动和恢复尾部。
- 生命周期：每种终止路径都恢复输入并关闭全部临时窗口。
- 扣子：未配置、成功、超时、服务错误和隐私边界。

### 9.2 视觉回归

使用本机微信 4.1.11 实机参考截图建立普通工具栏、滚动初始态和滚动增长态的黄金图对照。重点检查选区颜色、边框、控制点、工具栏尺寸、间距、图标描边、阴影、预览比例和控制条锚点。

视觉回归允许操作系统字体抗锯齿和 WebView 渲染产生的小范围像素差异，但不能接受肉眼可见的布局、尺寸、颜色或图标风格偏差。

### 9.3 Windows 实机验收

在 Edge/Chrome、微信聊天、文件资源管理器、Windows 设置和 Electron 应用中测试普通截图及滚动截图。滚动输入覆盖鼠标滚轮、触控板、滚动条、Page Down 和方向键。

完成标准：

- 普通截图的整体观感、工具顺序和操作反馈接近微信 4.1.11；
- 图标风格统一，无字符占位或混搭资源；
- 长截图大预览、右侧缩略图和四按钮控制条行为与微信一致；
- 最终长图无明显重复、缺行、断层、错位或固定栏重复；
- 完成后仍可继续标注、马赛克、OCR、翻译、复制和保存。

## 10. 实施范围控制

本轮只重做截图工具本身，不加入截图历史库、录屏、云同步、账号系统或团队分享。钉住和平台分享属于微信工具栏既有行为，因此保留；无法跨平台一致实现的分享能力必须提供明确的复制降级路径。
