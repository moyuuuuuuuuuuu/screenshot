# WeChat 4.1.11 Screenshot Parity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the complete screenshot workflow to match Windows WeChat 4.1.11 while retaining a configurable `Alt + Shift + A` shortcut, Coze-backed cloud features, and cross-platform boundaries.

**Architecture:** Use one platform-independent capture-session state machine and focused React components for the WeChat-style overlay. Keep native window/input/capture responsibilities in Rust, replace stable-frame-only long capture with continuous motion tracking, and render long-capture previews in coordinated borderless Tauri windows.

**Tech Stack:** Rust stable MSVC, Tauri 2, Windows APIs, React 19, TypeScript strict mode, Canvas/PNG, Vitest, Rust unit tests

## Global Constraints

- Work directly on `main`, as explicitly requested; do not create a worktree or feature branch.
- Use TDD for every behavior change: write and observe the focused failing test before implementation.
- Use the locally captured Windows WeChat `4.1.11.55` states as the only visual reference.
- Default shortcut is exactly `Alt + Shift + A`; users may change it locally.
- All toolbar art is project-owned SVG using an approximately `18×18` view, `1.6px` stroke, rounded caps/joins, and consistent padding.
- Do not use WeChat names, marks, or copyrighted brand assets in the product.
- Screenshot pixels remain local unless the user explicitly invokes OCR, translation, or privacy redaction.
- Preserve recoverable image data on errors; every termination path restores overlay input and closes temporary windows.
- Keep limits of 200 accepted scrolling frames, 60,000 output pixels, and 120 seconds unless a later approved spec changes them.
- Windows verification commands run in a VS 2022 x64 developer prompt with `CARGO_BUILD_JOBS=1` and Rust tests use `-j 1`.

```ts
export type CaptureSession = Readonly<{
  mode: CaptureMode;
  selection: Rect | null;
  sourceUrl: string;
  scrollResult: Blob | null;
  service: null | 'ocr' | 'translate' | 'redact';
}>;

export function captureSessionReducer(state: CaptureSession, action: CaptureAction): CaptureSession {
  switch (action.type) {
    case 'selectionCommitted': return { ...state, mode: 'annotating', selection: action.rect };
    case 'scrollStarted': return { ...state, mode: 'scrolling' };
    case 'scrollCancelled': return { ...state, mode: 'annotating', scrollResult: null };
    case 'scrollEditRequested': return { ...state, mode: 'annotating', sourceUrl: action.imageUrl, selection: null };
    case 'serviceStarted': return { ...state, mode: 'serviceBusy', service: action.service };
    case 'serviceFinished': return { ...state, mode: 'annotating', service: null };
    default: return assertNever(action);
  }
}

function assertNever(value: never): never {
  throw new Error(`Unhandled capture action: ${JSON.stringify(value)}`);
}
```
